## General Guidelines

### Belief System Design
- Start with clear, falsifiable beliefs
- Use appropriate belief types for different scenarios
- Maintain consistent observation contexts

### Dialectic Sessions
- Keep interactions focused on single contexts
- Validate user inputs before updating beliefs
- Handle belief updates atomically

### Error Handling
- Implement proper error handling for all API calls
- Log errors for debugging
- Provide meaningful error messages to users
